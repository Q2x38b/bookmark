Bookmark URL Parameters

Pipe data into BMarks from browser extensions, Shortcuts, or automation tooling by sending query parameters to /. The capture modal opens with everything pre-filled, even if you still need to sign in.

⌘ + D • 0→1 capture workflow

Quick reference

Works everywhere

macOS share sheet, Raycast, Alfred, browser extensions, or curl.

Autodetects content

Links, color values, and notes are inferred instantly.

Safe fallback

Invalid payloads become plain text so nothing is lost.

Supported parameters

Parameter Description Example
content (required) URL, hex color (e.g. #ff00ff), rgb()/hsl() string, or plain text. ?content=https://example.com
title (optional) Custom label. Links auto-fetch their page title; colors use the literal code. &title=New%20Docs
group (optional) Group name or UUID. If the group doesn’t exist it’s ignored and you can choose one manually. &group=Research
new (optional) When present (e.g. new=1), the modal opens even if no other fields are sent. &new=1

Examples

/?content=https://example.com/docs&title=Docs&group=Design
/?content=%23ffcc00&group=Palette
/?new=1

Behavior notes

  • Parameters are consumed once and scrubbed from the URL.
  • Not signed in? Payloads are cached for 10 minutes then replayed after auth finishes.
  • Invalid URLs or color strings fall back to plain text and require a manual title.
